RAVE: Randomized Noise Shuffling for Fast and Consistent Video Editing with Diffusion Models — happy8825

Noise shuffling Strategy를 사용하여 제로샷으로 text 기반 비디오 편집하는 모델이다. 

바로 모델부터 살펴보자

주어진 비디오 VK={I1,…,IK}는 K개의 개별 프레임으로 구성되며, 특정 비디오 편집을 위한 텍스트 프롬프트 P가 주어졌다고 가정했을때, RAVE의 목표는 편집된 비디오 VK∗를 생성하는 것이다. 이 과정에서 원래 비디오는 프롬프트 P에 따라 변화하며, 원래의 동작과 의미 구조는 유지되어야한다.

각각의 video frame들은 1. Condition Preprocessor, 2. DDIM Inversion에 태워지며, 전처리되어서 잘 process된 grid들은 controlnet의 입력으로 들어간다.

 

논문의 주요 아이디어인 Grid Trick에 대해서 알아보자.

Grid Trick

Grid Trick이란?

Grid Trick은 video frame들을  grid 형식으로 배열하여 하나의 이미지처럼 취급하고, 하나의 이미지처럼 취급하기에 일관적으로 image생성/편집이 가능해진다. 

Process

  1. 입력 데이터를 grid로 변환: 비디오의 K개의 프레임을 n×m 형태의 grid G로 배열한다. Grid: G(1,…,N)∈R(W⋅m)×(H⋅n)×3 :
    • W,H: 각 프레임의 너비와 높이.
    • n,m: 그리드의 행(row)과 열(column) 개수.
    • N=n⋅m: 그리드에 포함된 총 프레임 수.
    •  (1,…,N): 그리드 내 프레임의 순서.
  2. Diffusion 모델에 입력: Grid를  Diffusion에 입력한다. 모델은 G를 단일 이미지처럼 처리하여 전체 프레임에 일관된 스타일을 적용할 수 있다.
  3. 수정된 grid를 다시 분리: 수정된 그리드를 원래의 개별 프레임으로 나누고, 이 프레임들을 순차적으로 연결하여 최종 비디오를 생성한다.

그치만 여기까지하면, GPU등에 따라서 한 grid에 올릴 수 있는 frame의 수에 제한이 생길것이고, 그럼 나눠서 processing하다 보면 일관성 유지가 쉽지않을것이다. 그래서 또 제안된것이 Noise shuffling이다.

Noise Shuffling

 

기존의 Grid Trick은 동일한 그리드 내에서는 프레임 일관성을 유지할 수 있지만, 그리드 간에는 시간적 일관성이 깨질 가능성이 있었다. Noise Shuffling은 각 denoising step에서 프레임 순서를 무작위로 섞는 방식을 통해 이 문제를 해결한다.

Process

  1. Grid 생성: 입력 비디오 VK를 n×m 형태의 그리드 G로 변환한다. 그리드는 L개의 독립적인 블록으로 나뉘며, 각 블록은 다음과 같이 표현된다: video2grid(VK,N)={G1(1,…,N),G2(N+1,…,2N),…,GL(P,…,K)} 여기서:
    • L=K/N: 생성된 그리드의 총 개수
    • N=n×m: 각 그리드의 총 프레임 수
    • P=K−(N+1): 각 그리드의 마지막 프레임 인덱스
  2. 랜덤 셔플링: 각 denoising step에서, 모든 그리드의 프레임 순서를 무작위로 섞는다: shuffle(GL)={G1(i,…,j),G2(k,…,l),…,GL(x,…,y)} 여기서 (i,…,j),(k,…,l),(x,…,y)는 무작위로 선택된 프레임 인덱스
  3. 랜덤하게 섞인 프레임은 각 디노이징 단계에서 spatio-temporal interaction 이 강화되어 시간적 일관성을 유지할수있다.

Noise Shuffling은 프레임 간 interaction을 강화하기 위해 다음 두 가지 메커니즘을 사용한다

  1. spatio-temporal interaction: 각 디노이징 단계에서 랜덤 순서를 사용하면, 다른 프레임의 정보가 latent vector에 영향을 미친다. 이는 pixel shifts를 줄이고,flickering을 감소시킨다.
  2. self-attention layer:  각 프레임이 전체 스타일의 영향을 받도록 보장한다. 따라서, 독립적인 그리드 간에도 스타일이 일관되게 적용된다.

Denoising Step

 Noise Shuffling은 디노이징 step에서 프레임 순서를 무작위로 변경한다. 디노이징 step에서 t-번째 잠재 벡터는 다음과 같이 정의된다: {zt(1),zt(2),…,zt(N)} 여기서:

  • zt(i): i-번째 프레임의 t-단계 latent vector

Noise Shuffling은 이 벡터의 순서를 무작위로 섞어 다음 상태로 전달한다: shuffle({zt(1),zt(2),…,zt(N)})={zt(k),zt(j),…,zt(i)}

  1. latent 벡터 간 상호작용: 랜덤 셔플링을 통해 각 프레임의 latent vector는 다른 프레임의 영향을 받습니다. 이는 다음과 같이 나타낼 수 있다: zt+1(i)=f(zt(j),zt(k),…)for j,k∈shuffle(indices) 여기서 f는 latent 벡터 간 상호작용을 나타내는 함수
  2. Self attention layer : 랜덤화된 순서는 self attention layer에서 다음과 같은 style tranfser 이 가능함: S(zt(i))=∑j=1Nwij⋅zt(j) 여기서 wij는 i-번째 프레임과 j-번째 프레임 간의 attention 가중치.

Results:

 

Consistency도 잘 유지되는듯 하고

극단적인 editing도 가능한것같다.

사실 grid 기반이라 어쨋거나 grid가 나눠지는거니까 비디오가 길어지면 아무리 노이즈를 섞어도 consistency를 유지하기는 쉽지않을것같음. 역시나 논문에 한계점으로 27frame이 넘어가면 flickering이 발생한다고 한다. 하지만 제로샷으로 이정도면 아주 놀라운것 같다. 뭔가 consistency 유지하기 위해 first frame혹은 이전 frame을 conditioning하는 논문들을 많이 봐왔는데, grid approach는 되게 novel한것 같다. 

Ablation Study

Shuffling을 해주는게 확실히 consistency에 도움이 되는것같다. DDIM Inv이 빠진 row를 봤을때, DDIM inv는 빠졌지만 shuffling이 있어서 frame별 consistency는 유지가 되는 반면 Shuffling이 빠져버리면, 아예 consistency가 꺠져버리는것 같다.

 

 

Shuffling말고, Grid approach인데 맨 처음 Grid 1x1열 은 모두 첫번째 frame을 고정시키는거 vs shuffling 뭐가 더 효과가 좋을지도 실험해보고싶다.
