Flow matching guide and code 7탄 — happy8825

이번 포스트부터는 Flow Matching을 본격적으로 알아보겠다. 간단한 분포에서 시작해 복잡한 타겟 분포로 부드럽게 연결해주는 확률 경로와, 이를 생성하는 velocity field를 학습하는 것이 목표이다.

Flow Matching의 목표

FM의 목표는 다음과 같이 요약된다.

  • Source 분포 p 와 target 분포 q 가 주어졌을 때, 시간 t∈[0,1] 에 따라 분포가 p0=p 에서 p1=q 로 연속적으로 변하는 probability path pt 를 정의한다.
  • 그리고 이 probabiltiy path를 만들어내는 velocity field utθ 를 학습한다.
  • 즉, p 에서 q 로 가는 flow를 한 번에 확률적으로 연결하지 않고, 작은 변화들의 연속으로 보고 그 변화를 일으키는 velocity field를 학습한다고 볼 수 있다.

Find utθ generating pt,p0=p,p1=q.

여기서 “generating”이란, t=0 에서 p0=p 이고 t=1 에서 p1=q 가 되도록, 미분방정식을 통해 probabiltiy path를 만들어낸다는 의미이다.

아주 이해하기 좋게 잘 만들어진 그림인거 같다. Flow matching의 boudnary condition은 시점 0 에서는 p여야하고, 시점 1에서는 실제 데이터 distribution q여야한다는 boundary condition이 있는데, 이걸 이렇게 분포대 분포로 다 맞추지 말고,

샘플 하나를 잡아서, 그 점에 대한 boundary condition을 만족하게 하고, 이걸 다 합쳐서 분포를 맞추자는게 가장 큰 핵심 아이디어인거 같다.

Probability path 정의, velocity field 학습

  1. source 분포 p 와 타겟 분포 q 를 정의한다. (p 는 simple한 가우시안, q 는 이미지 분포 등)
  2. 확률 경로 pt 를 정의한다.
  3. 이 경로 pt 를 만족하도록 하는 velocity field ut 를 neural network(파라미터 θ)로 근사한다. 즉, utθ 라는 모델이 실제 velocity field ut 를 잘 흉내내도록 학습한다.
  4. 학습이 끝나면, p 에서 샘플링한 점들을 ODE solver등으로 q 로 보낼 수 있다. 이 과정을 통해 실제로 q 샘플(데이터)을 생성할 수 있다.

Loss Funciton

FM에서는 gt 속도” ut(Xt) 와 “모델이 예측한 속도” utθ(Xt) 간의 차이를 줄이는 loss를 사용한다:

LFM(θ)=EXt∼pt[D(ut(Xt),utθ(Xt))].

즉, pt 에서 샘플링된 Xt 에 대해 모델의 속도 예측이 실제 속도와 가깝도록 regression하는 방식이다.

Data

  • Source: X0∼p. (가우시안 분포에서 뽑은 noisy 벡터)
  • Target: X1∼q. (실제 이미지, 오디오 등 복잡한 데이터 분포)

Coupling?

  • 만약 X0 와 X1 이 서로 독립이라면, π0,1(x0,x1)=p(x0)q(x1) 로 표현할 수 있다.
  • 어떤 의존 관계를 갖는 경우(저해상도 → 고해상도 변환), π0,1 는 더 일반적인 coupling 분포가 된다.

Conditional Probability Path pt∣1(x∣x1)

FM에서는 conditional strategy를 통해 복잡한 전체 경로를 “타겟 샘플 x1 기준으로 조각조각” 구성한다. 즉, “타겟이 x1 일 때, x 가 시간에 따라 어떻게 변하는가?”를 나타내는 pt∣1(x∣x1) 를 정의한다.

그리고 나서 x1 에 대해 q(x1) 로 적분하면 최종적인 marginal probabitiy path pt(x) 를 얻는다:

pt(x)=∫pt∣1(x∣x1)q(x1)dx1.

Boundary Conditions

FM에서 가장 중요한 조건은 p0=p , p1=q 를 만족하는 것이다.

이를 “conditional 한 관점”에서 보면, 각 타겟 샘플 x1 에 대해 시작(t=0)과 끝(t=1)을 아래처럼 고정한다:

p0∣1(x∣x1)=π0∣1(x∣x1),p1∣1(x∣x1)=δx1(x).

  • p0∣1(x∣x1)=π0∣1(x∣x1): 시간 t=0 에서는, 미리 정해둔 joint distribution π0,1 (또는 그 조건부 π0∣1)과 동일하게 둔다.
  • p1∣1(x∣x1)=δx1(x): 시간 t=1 에서는 x1 에 확률 질량이 100% 모이는 delta 분포가 되도록 만든다. 즉, 최종적으로 x 가 x1 로 완전히 수렴한다는 의미다.

이렇게 설정하면, 적분을 통해 얻어지는 pt 또한 p0=p , p1=q 를 만족하게 된다.

Conditional Velocity Field

그럼 marginal probabiltiy path pt”를 만들어내는 velocity field ut를 어떻게 구성할까. 핵심 아이디어는, “condtiional velocity field”들을 모아서 marginal velocity field 를 얻는다는 것이다.

 

ut(x) 는 “x라는 현재 상태에서, x1이었다고 가정한 velocity field”을 “x1이었을 확률”로 평균낸 것과 동일하다.

이거를 기댓값 형태로도 다음과 같이 풀어서 나타낼 수 있다.

그러면 ut(x)=E[ut(Xt∣X1)|Xt=x] 인 것은, “Xt=x에서의 속도”를 ‘condition (X1)’에 대해 평균 낸 값이라는 말이다. 이는 곧, conditional velocity field를 통합해 가장 가깝게 Xt=x일 때의 실제 속도를 재현한다는 뜻이다.

 

정리하자면

  1. 조건부 속도장 ut(⋅∣x1) 은 “X1=x1”라고 가정했을 때의 probability path를 만든다.
  2. 주변 속도장 ut(x) 는, 이러한 conditional velocity field들을 posterior 가중치 p1∣t(x1∣x)로 평균 낸 형태다.
  3. 이를 통해 시점 t에서 Xt=x가 되는 샘플은, 여러 x1 중 어떤 것을 타겟으로 했는지 모름에도 ‘평균적’으로 위에서 정의한 ut(x)를 따라간다.
  4. 결국 pt(x)를 만들어내는 velocity field ut(x)가 자연스럽게 유도되며, 이는 conditional velocity field의 합이다.
