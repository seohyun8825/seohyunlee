Towards Open-Vocabulary Semantic Segmentation Without Semantic Labels — happy8825

<Intro>

CLIP 같은 비전-언어 모델은 이미지에 어떤 객체가 있는지 알아내는 데는 뛰어나지만, 픽셀 단위로 객체의 위치를 찾는 작업은 아직 잘 못한다. 이를 해결하기 위해 PixelCLIP을 제안하는 논문이다. 이 방법은 SAM과 DINO 같은 모델로 생성한 마스크와 라벨 없는 이미지를 활용해 CLIP을 픽셀 수준으로 확장한다. 특히, 마스크에 label이 없는 문제를 해결하기 위해 online clustering 알고리즘을 적용해 genral한 의미적인 개념을 학습한다.

 

Open-Vocabulary Semantic Segmentation 이란?

이미지의 각 픽셀에 대해 클래스 레이블을 예측하는 task. 하지만 이 작업에 필요한 데이터셋은 픽셀 단위의 세밀한 annotation 작업이 필요함 --> 이를 해결하기 위해, 최근에는 CLIP이나 ALIGN과 같은 Vision-Language 모델을 사용해서, Open-Vocabulary Semantic Segmentation (즉, 제한 없는 다양한 클래스에 대한 추론)을 가능하게 하는 방법들이 연구되고 있다.

 

기존 방법의 한계점

  • 픽셀 수준의 레이블 필요
  • 이미지 캡션의 한계
    일부 연구에서는 이미지 캡션(예: "강아지가 있는 사진")과 같은 이미지 레벨 레이블을 활용하지만, 이는 객체가 "어디에 있는지"에 대한 공간적 정보를 제공하지 못함.
  • 성능의 제한
    CLIP과 같은 모델은 객체가 무엇인지(what)를 잘 인식하지만, 객체가 어디에 있는지(where)를 명확히 학습하지 못함.

PixelClip의 핵심 아이디어

픽셀 레이블 없이도 CLIP을 활용한 open vocapulary semantic segmentation을 다음과 같은 framework를 통해 가능하게함.

  • 객체의 위치 정보 제공 
    • CLIP이 "어디를 봐야 하는지"를 학습하도록, DINO나 SAM을 활용해 이미지를 세밀한 영역으로 나눈다. 이렇게 생성된 마스크를 활용해 CLIP의 이미지 인코더를 finetuning
  • Clip의 text 인코더
    • prompt learning을 통해 learnable class 생성 -> 온라인 클러스팅 알고리즘으로 유사한 마스크를 의미적으로 그룹화하는데 사용

 

<Method>

모델의 입력

  • 이미지 I∈RH×W×3: H×W 크기의 RGB 이미지
  • 텍스트 클래스 집합 T={Ti}i=1S: 이미지와 관련된 S개의 텍스트 설명

모델의 출력

  • 이미지-텍스트 유사도 맵 MIT∈Rh×w×S: h×w 해상도의 각 픽셀에서 S개의 텍스트 클래스와의 유사도를 나타내는 맵.

대략적인 모델 구조

  1. 이미지 피쳐맵 생성: CLIP의 비전 인코더 EV를 통해 입력 이미지 I에서 특징 맵 fI를 추출한다. fI=EV(I)∈Rh×w×d 
  2. 텍스트 피쳐 생성: CLIP의 텍스트 인코더 EL를 통해 텍스트 클래스 집합 T의 특징 fT를 생성: fT=EL(T)∈RS×d 여기서 S는 클래스의 개수
  3. image - text 유사도 계산: 이미지 피쳐 fI(x,y)와 텍스트 피쳐 fT(n) 간의 코사인 유사도를 계산하여 유사도 맵 MIT를 생성한다: MIT(x,y,n)=fI(x,y)⋅fT(n)‖fI(x,y)‖‖fT(n)‖ 이 수식을 통해 픽셀 (x,y)가 클래스 Tn과 얼마나 유사한지를 나타냄. 

즉,  픽셀 수준에서 클래스와의 유사도를 예측하는 능력을 학습을 하는데, 이를 위해, 이미지-텍스트 similarity map MIT을 기반으로  Lmask를 최소화한다.

Integrating masks into Clip Features

마스크 M은 DINO나 SAM과 같은 사전 학습된 비전 모델을 통해 생성되지만, 이 마스크는 semantic label을 포함하지 않는다는 문제가 있었음. 이를 해결하기 위해

1. 마스크 M을 사용하여 CLIP 이미지 인코더로부터 생성된 이미지 feature fI에서 mask feature 추출

fM=MaskPool(fI,M)∈RN×d

2. 이미지 feature fI와 mask feature fM 간의 유사도를 계산하여 MIM를 생성

MIM(x,y,n)=fI(x,y)⋅fM(n)‖fI(x,y)‖‖fM(n)‖

3. MIM와 마스크 M 간의 유사도를 기반으로  Lmask를 정의하여 모델을 학습한다. 추가로 해상도 문제 해결하기 위해 mask decoder 를 하나 더 달았음.

즉, objective는 다음과 같음.

M~=D(MIM)

Semantic Clustering of Masks

해결해야할 문제가 하나 더 있었는데, DINO와 SAM이 생성한 mask가 Over-segmentation 문제를 일으켜, 의미론적으로 같은 영역을 다 쪼개서 구분해버리는 문제가 있었음. 마스크를 너무 많이 예측을 한 case.

이걸 해결하기 위해, 의미론적으로 유사한 mask를 global cluster로 그룹화해서 학습을 단순화한다.

우선 목표는, DINO나 SAM에서 생성된 mask들을 의미론적으로 유사한 카테고리로 clustering을 하는것이다.

프롬프트 토큰 : C∈Rk×l×de

클래스 feature: fC=EL(P∗,C)∈Rk×d 

 

그럼 앞서 나왔던 mask feature이랑ㄴ class feature이랑 매칭을 시켜주기 위해 Q 행렬을 정의한다

 

Q 행렬: Q∈R+k×m 

 

그리고 이걸 최적화하는데, 일단 목표는 마스크 feature이랑 Class Feature이랑 유사성이 최대화되면 된다.

이 목표를 최적화 문제로 정의해보면 다음 식과 같다.

maxQ∈R+k×mTr(Q⊤FM⊤fC)+ϵH(Q),s.t. Q⊤1k=1m1m,Q1m=1k1k

 

즉 Q행렬을 직관적으로 해석해보자면, Q는 마스크와 클러스터 간의 soft assignment를 나타내며, Qij는 마스크 j가 클러스터 i에 속할 확률.

Q행렬 관련 최적화 문제를 잘 풀다 보면, fc(i)에 대한 식을 얻을 수 있고,  그럼 fc(i) 와  이미지 feature의 similarity 맵을 구할 수 있음.
